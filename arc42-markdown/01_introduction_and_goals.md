# 1. Introducción y Objetivos

## 1.1 Objetivo del Sistema ERP

El Sistema ERP tiene como objetivo integrar y automatizar los principales procesos de negocio de la empresa, permitiendo una gestión centralizada, eficiente y confiable de la información.  
El sistema busca mejorar la toma de decisiones, reducir errores operativos y garantizar la trazabilidad de las operaciones relacionadas con compras, inventarios, facturación, activos, empleados y análisis gerencial.

Para este taller, el enfoque principal se centra en el **Módulo de Compras**, el cual constituye una funcionalidad crítica para el control de proveedores, productos y órdenes de adquisición.

---

## 1.2 Requisitos de Negocio – Módulo de Compras

Los principales requisitos de negocio identificados para el Módulo de Compras son los siguientes:

- El sistema debe permitir el **registro y gestión de productos**, incluyendo su información básica.
- El sistema debe permitir el **registro y administración de proveedores**, con sus datos de contacto.
- El sistema debe permitir la **asociación de productos con proveedores**, incluyendo precios de compra.
- El sistema debe permitir la **creación y gestión de órdenes de compra**.
- El sistema debe permitir la **consulta del historial de compras** realizadas.
- El sistema debe garantizar la **integridad y validación de la información** ingresada.
- El sistema debe proporcionar **trazabilidad de las operaciones de compra** para auditoría y control.

Estos requisitos constituyen la base para el diseño de la arquitectura del sistema y para la definición del backlog del producto.
